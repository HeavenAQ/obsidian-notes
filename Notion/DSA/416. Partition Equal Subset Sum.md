## Summary

Given an integer array `nums`, decide whether it can be split into two subsets whose sums are equal. Every element must go to exactly one of the two subsets.

**Constraints:** `1 <= nums.length <= 200`; `1 <= nums[i] <= 100`.

**Examples:**

- `nums = [1,5,11,5]` → `true` (split into `[1,5,5]` and `[11]`, both sum to 11)
- `nums = [1,2,3,5]` → `false` (total is 11, which is odd — no equal split exists)

## Hints

1. If the total sum is odd, you can answer immediately. Otherwise, what single-subset question is this equivalent to?
2. The problem reduces to: can some subset of `nums` sum to exactly `total / 2`? That's a classic **0/1 knapsack (subset-sum)** question.
3. Define `dp[j]` = "can some subset of the numbers processed so far sum to exactly `j`?" Start with `dp[0] = true` and process numbers one at a time.
4. When updating the 1D `dp` array for a new number `num`, iterate `j` **from **`**target**`** down to **`**num**` — going downward prevents using the same number twice in one subset.

### Approach / Intuition

Compute the total sum. If it's odd, return `false` right away. Otherwise set `target = sum / 2` and solve subset-sum: does any subset reach exactly `target`? If one subset sums to `target`, the remaining elements automatically sum to the other half.

Use a boolean array `dp` of size `target + 1`, where `dp[j]` means "a subset summing to `j` is achievable with the numbers considered so far". For each number `num`, update `dp[j] = dp[j] || dp[j - num]`, iterating `j` from high to low so each number is used at most once (the 0/1 knapsack trick — iterating upward would let `dp[j - num]` already include `num`, i.e. unbounded reuse).

**Time complexity:** O(n · target), where `target = sum / 2` (at most 10,000 given the constraints).

**Space complexity:** O(target)

```java
class Solution {
    public boolean canPartition(int[] nums) {
        int sum = 0;
        for (int num : nums) {
            sum += num;
        }
        if (sum % 2 != 0) {
            return false;
        }
        int target = sum / 2;
        boolean[] dp = new boolean[target + 1];
        dp[0] = true; // empty subset sums to 0
        for (int num : nums) {
            for (int j = target; j >= num; j--) {
                dp[j] = dp[j] || dp[j - num];
            }
            if (dp[target]) {
                return true; // early exit
            }
        }
        return dp[target];
    }
}
```

**Why this works:** each `dp` update considers exactly two choices for the current number — skip it (keep `dp[j]`) or take it (use `dp[j - num]` from the state *before* this number was processed, guaranteed by the downward iteration). So after all numbers are processed, `dp[target]` is true iff some subset sums to exactly half the total.
