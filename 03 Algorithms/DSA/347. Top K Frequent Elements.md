## Summary

Given an integer array `nums` and an integer `k`, return the `k` most frequent elements. You may return the answer in any order.

Constraints:
- `1 <= nums.length <= 10^5`
- `k` is in the range `[1, number of distinct elements in nums]`
- It is guaranteed that the answer is unique

Follow-up: your algorithm's time complexity must be better than $O(n \log n)$, where $n$ is the array's size.

**Example 1**
Input: `nums = [1,1,1,2,2,3]`, `k = 2`
Output: `[1,2]`

**Example 2**
Input: `nums = [1]`, `k = 1`
Output: `[1]`

## Hints

1. Start by counting how often each value appears.
2. Sorting all distinct values by frequency costs $O(n \log n)$ — the follow-up wants better than that. What bound do you know on the *frequency* values themselves?
3. Frequency of any element can range only from `1` to `n` (the array length). Could you use frequency itself as an array index (bucket)?
4. Build an array of buckets where `bucket[f]` holds all elements whose count is exactly `f`, then walk the buckets from high frequency to low, collecting elements until you have `k`.

> [!note]- Solution (click to reveal)
> **Approach: Bucket Sort by Frequency**
>
> 1. Count each number's frequency with a `HashMap<Integer, Integer>`.
> 2. Since frequency can only range from `1` to `n` (array length), create `buckets`, an array of `List<Integer>` of size `n + 1`, where `buckets[f]` collects every number whose frequency equals `f`.
> 3. Walk `buckets` from index `n` down to `1`. For each number encountered, add it to the result until the result has `k` elements.
>
> This avoids sorting entirely — every element is placed into exactly one bucket, and we scan the buckets at most once.
>
> **Why this works:** because frequency is bounded by `n`, bucket indices never exceed `n`, so we get a valid counting/bucket sort in linear time instead of comparison-based sorting.
>
> **Complexity:** Time $O(n)$ — one pass to count, one pass to bucket, one pass to collect. Space $O(n)$ for the map and buckets.
>
> ```java
> import java.util.*;
>
> class Solution {
>     public int[] topKFrequent(int[] nums, int k) {
>         Map<Integer, Integer> countMap = new HashMap<>();
>         for (int num : nums) {
>             countMap.merge(num, 1, Integer::sum);
>         }
>
>         // buckets[f] = list of numbers that occur exactly f times
>         List<Integer>[] buckets = new List[nums.length + 1];
>         for (Map.Entry<Integer, Integer> entry : countMap.entrySet()) {
>             int num = entry.getKey();
>             int freq = entry.getValue();
>             if (buckets[freq] == null) {
>                 buckets[freq] = new ArrayList<>();
>             }
>             buckets[freq].add(num);
>         }
>
>         int[] result = new int[k];
>         int idx = 0;
>         for (int freq = buckets.length - 1; freq >= 1 && idx < k; freq--) {
>             if (buckets[freq] == null) continue;
>             for (int num : buckets[freq]) {
>                 if (idx == k) break;
>                 result[idx++] = num;
>             }
>         }
>
>         return result;
>     }
> }
> ```
>
> **Alternative:** a min-heap of size `k` (`PriorityQueue` ordered by frequency) gives $O(n \log k)$, which is also acceptable and simpler to write if bucket sort feels unfamiliar.
