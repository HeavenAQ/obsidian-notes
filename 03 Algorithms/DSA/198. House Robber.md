## Summary

You are a robber planning to rob houses along a street. Each house `i` has `nums[i]` money. Adjacent houses have connected alarms, so you cannot rob two adjacent houses. Return the maximum amount you can rob without alerting the police.

**Constraints:** `1 <= nums.length <= 100`, `0 <= nums[i] <= 400`.

**Example 1:** `nums = [1,2,3,1]` → `4` (rob house 0 and house 2: 1 + 3).

**Example 2:** `nums = [2,7,9,3,1]` → `12` (rob houses 0, 2, 4: 2 + 9 + 1).

## Hints

1. For each house you have a binary choice: rob it or skip it. What does robbing house `i` prevent you from doing?
2. Define `dp[i]` = best total considering only the first `i+1` houses. Express `dp[i]` using `dp[i-1]` and `dp[i-2]`.
3. You only ever look two steps back — can you drop the array and keep just two variables?

+++ Solution (click to reveal)

**Approach.** Classic 1D DP. Let `dp[i]` be the max loot from houses `0..i`. Either skip house `i` (take `dp[i-1]`) or rob it (take `dp[i-2] + nums[i]`):

`dp[i] = max(dp[i-1], dp[i-2] + nums[i])`

Since only the previous two states matter, roll the array into two variables.

**Complexity.** Time `O(n)`, space `O(1)`.

```java
class Solution {
    public int rob(int[] nums) {
        int prev2 = 0; // dp[i-2]
        int prev1 = 0; // dp[i-1]
        for (int num : nums) {
            int cur = Math.max(prev1, prev2 + num);
            prev2 = prev1;
            prev1 = cur;
        }
        return prev1;
    }
}
```

**Why this works.** Any optimal plan for houses `0..i` either uses house `i` or it doesn't; both branches reduce to optimal subproblems on shorter prefixes, so the recurrence covers every legal plan exactly once (optimal substructure, no aftereffects).
