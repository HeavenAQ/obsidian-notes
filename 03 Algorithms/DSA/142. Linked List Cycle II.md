## Summary

Given the `head` of a linked list, return the node where the cycle begins. If there is no cycle, return `null`.

There is a cycle in a linked list if some node in the list can be reached again by continuously following the `next` pointer. Internally, `pos` denotes the index of the node that the tail's `next` pointer connects to (0-indexed), and `pos = -1` if there is no cycle. `pos` is **not** passed as a parameter — it's only there to describe the test case. You must **not modify** the linked list.

Constraints:
- The number of nodes in the list is in the range $[0, 10^4]$.
- $-10^5 \le \text{Node.val} \le 10^5$
- `pos` is `-1` or a valid index in the linked list.

Example 1: `head = [3,2,0,-4]`, `pos = 1` → output: the node with value `2` (tail connects to index 1).

Example 2: `head = [1,2]`, `pos = 0` → output: the node with value `1`.

Example 3: `head = [1]`, `pos = -1` → output: `null` (no cycle).

## Hints

1. Can you solve it using $O(1)$ extra space, i.e. without a hash set of visited nodes?
2. Floyd's Tortoise and Hare (a slow pointer moving 1 step and a fast pointer moving 2 steps) can detect whether a cycle exists at all.
3. Once the two pointers meet inside the cycle, think about the relationship between the distance from `head` to the cycle's start and the distance from the meeting point back to the cycle's start.
4. After the first meeting point is found, reset one pointer to `head` and advance both pointers one step at a time — they will meet exactly at the start of the cycle.

> [!note]- Solution (click to reveal)
> **Approach: Floyd's Tortoise and Hare (cycle detection + entry point)**
>
> Phase 1 — detect a cycle: move `slow` one step and `fast` two steps at a time. If they meet, a cycle exists. If `fast` reaches `null`, there is no cycle.
>
> Phase 2 — find the entry point: let the distance from `head` to the cycle start be $a$, from the cycle start to the meeting point be $b$, and the remaining cycle length back to the start be $c$ (so the cycle length is $b + c$).
>
> When `slow` and `fast` meet:
> - `slow` has traveled $a + b$.
> - `fast` has traveled $a + b + k(b + c)$ for some integer $k \ge 1$ (it has looped the cycle $k$ extra times), and since `fast` moves twice as fast, $2(a+b) = a + b + k(b+c)$, which simplifies to:
>
> $$a = k(b + c) - b$$
>
> Taking $k = 1$: $a = c$. This means the distance from `head` to the cycle start equals the distance from the meeting point to the cycle start (going forward around the cycle). So resetting one pointer to `head` and advancing both one step at a time makes them meet exactly at the cycle's entry node.
>
> **Complexity:** Time $O(n)$, Space $O(1)$.
>
> ```java
> /**
>  * Definition for singly-linked list.
>  * class ListNode {
>  *     int val;
>  *     ListNode next;
>  *     ListNode(int x) {
>  *         val = x;
>  *         next = null;
>  *     }
>  * }
>  */
> public class Solution {
>     public ListNode detectCycle(ListNode head) {
>         ListNode slow = head;
>         ListNode fast = head;
>
>         // Phase 1: detect whether a cycle exists
>         while (fast != null && fast.next != null) {
>             slow = slow.next;
>             fast = fast.next.next;
>
>             if (slow == fast) {
>                 // Phase 2: find the entry point of the cycle
>                 ListNode ptr = head;
>                 while (ptr != slow) {
>                     ptr = ptr.next;
>                     slow = slow.next;
>                 }
>                 return ptr;
>             }
>         }
>
>         return null; // no cycle
>     }
> }
> ```
