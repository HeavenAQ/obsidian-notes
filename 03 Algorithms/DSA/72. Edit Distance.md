## Summary

You're given two strings, `word1` and `word2`. Return the minimum number of operations needed to convert `word1` into `word2`, where each operation is one of:

1. Insert a character
2. Delete a character
3. Replace a character

**Constraints:** `0 <= word1.length, word2.length <= 500`; both strings consist of lowercase English letters.

**Example 1:**

```javascript
Input: word1 = "horse", word2 = "ros"
Output: 3
Explanation: horse -> rorse (replace 'h' with 'r') -> rose (remove 'r') -> ros (remove 'e')
```

**Example 2:**

```javascript
Input: word1 = "intention", word2 = "execution"
Output: 5
```

## Hints

4. Compare the *last* characters of `word1` and `word2` first. If they match, they don't cost anything — the problem reduces to the same question on the two shorter prefixes.
5. If they don't match, you have exactly three choices: insert a character into `word1` (to match word2's last char), delete `word1`'s last character, or replace it. Each choice reduces the problem to a smaller subproblem — which one?
6. Define `dp[i][j]` = edit distance between `word1[0..i)` and `word2[0..j)`. Base cases: `dp[i][0] = i` (delete everything), `dp[0][j] = j` (insert everything). Fill the table bottom-up in O(m·n) time and O(m·n) space (optimizable to O(n)).

> [!note]+ Solution (click to reveal)
> **Approach / intuition:** This is the classic Edit Distance (Levenshtein Distance) problem, solved with 2D bottom-up dynamic programming.
> 
> Let `dp[i][j]` be the minimum number of operations to convert the first `i` characters of `word1` into the first `j` characters of `word2`.
> 
> - Base cases: `dp[i][0] = i` and `dp[0][j] = j`.
> - Transition: if `word1[i-1] == word2[j-1]`, the characters already match, so `dp[i][j] = dp[i-1][j-1]` (no extra cost).
> - Otherwise, take the best of the three operations and add 1: `dp[i][j] = 1 + min(dp[i-1][j-1] /* replace */, dp[i-1][j] /* delete from word1 */, dp[i][j-1] /* insert into word1 */)`.
> 
> The answer is `dp[m][n]`.
> 
> **Why this works:** every edit sequence that transforms word1 into word2 can be decomposed into a sequence of these three primitive operations applied to some prefix, and the DP explores all ways of aligning the two strings optimally, which is exactly what the recurrence captures.
> 
> **Complexity:** Time O(m·n), Space O(m·n) — can be reduced to O(min(m,n)) by keeping only two rows, but the full table is shown here for clarity.
> 
> ```java
> class Solution {
>     public int minDistance(String word1, String word2) {
>         int m = word1.length();
>         int n = word2.length();
>         int[][] dp = new int[m + 1][n + 1];
> 
>         for (int i = 0; i <= m; i++) {
>             dp[i][0] = i;
>         }
>         for (int j = 0; j <= n; j++) {
>             dp[0][j] = j;
>         }
> 
>         for (int i = 1; i <= m; i++) {
>             for (int j = 1; j <= n; j++) {
>                 if (word1.charAt(i - 1) == word2.charAt(j - 1)) {
>                     dp[i][j] = dp[i - 1][j - 1];
>                 } else {
>                     int replace = dp[i - 1][j - 1];
>                     int delete = dp[i - 1][j];
>                     int insert = dp[i][j - 1];
>                     dp[i][j] = 1 + Math.min(replace, Math.min(delete, insert));
>                 }
>             }
>         }
> 
>         return dp[m][n];
>     }
> }
> ```
