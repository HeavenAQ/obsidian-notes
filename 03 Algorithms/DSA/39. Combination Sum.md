## Summary

Given an array of **distinct** positive integers `candidates` and a target integer `target`, return a list of all **unique combinations** of `candidates` where the chosen numbers sum to `target`. The **same number may be chosen from `candidates` an unlimited number of times**. Two combinations are unique if the frequency of at least one of the chosen numbers is different.

Constraints:
- `1 <= candidates.length <= 30`
- `2 <= candidates[i] <= 40`
- All elements of `candidates` are distinct.
- `1 <= target <= 40`

**Example 1:**
Input: `candidates = [2,3,6,7]`, `target = 7`
Output: `[[2,2,3],[7]]`
Explanation: `2+2+3 = 7` and `7 = 7`. These are the only two combinations.

**Example 2:**
Input: `candidates = [2,3,5]`, `target = 8`
Output: `[[2,2,2,2],[2,3,3],[3,5]]`

## Hints

1. This is a classic backtracking / DFS problem: at each step you either "use the current candidate again" or "move to the next candidate."
2. Sort the candidates first — it lets you prune the search early once the running sum exceeds `target`, and it makes it easy to skip over values that can never work.
3. To avoid duplicate combinations (like `[2,2,3]` vs `[2,3,2]`), only allow picking candidates at index `>= current index` in your recursive calls — never go backwards.
4. Since a number can be reused, when you recurse after picking `candidates[i]`, pass `i` again (not `i+1`) as the next allowed starting index.
5. Think about the state you need to track: a running sum (or remaining target), the current index, and the current path — backtrack (remove the last element) after exploring each branch.

> [!note]- Solution (click to reveal)
> **Approach:** Classic backtracking with pruning. Sort `candidates` ascending first. Do a DFS where at each call we track the remaining target and the index we're allowed to start picking from (`start`). At each node:
> - If `remaining == 0`, record the current path as a valid combination.
> - Otherwise, iterate `i` from `start` to the end of the array. If `candidates[i] > remaining`, break immediately (sorted array means all later candidates are also too big — this is the key pruning step).
> - Otherwise, add `candidates[i]` to the path, recurse with `remaining - candidates[i]` and `start = i` (reuse allowed), then remove it from the path (backtrack).
>
> **Why this works:** Passing `start = i` (rather than `i + 1`) into the recursive call allows the same number to be reused arbitrarily many times, since future picks can still choose index `i`. Forcing `i >= start` at every level (never going back to an earlier index) ensures each combination is generated in non-decreasing order of value, which eliminates duplicate permutations of the same multiset (e.g. `[2,2,3]` is only ever built as `2,2,3`, never `2,3,2`). Sorting up front turns the "skip candidates that are too big" check into an early `break`, which prunes large parts of the search space.
>
> **Complexity:** Sorting is $O(n \log n)$. The search space is bounded by the number of valid combinations, which in the worst case is exponential in `target / min(candidates)`; a loose but standard bound is $O(2^{target})$ time in the worst case, with $O(target / min(candidates))$ recursion depth for the call stack / current path (space also holds the output itself, which dominates in practice).
>
> ```java
> import java.util.ArrayList;
> import java.util.Arrays;
> import java.util.List;
>
> class Solution {
>     public List<List<Integer>> combinationSum(int[] candidates, int target) {
>         Arrays.sort(candidates);
>         List<List<Integer>> result = new ArrayList<>();
>         List<Integer> path = new ArrayList<>();
>         backtrack(candidates, target, 0, path, result);
>         return result;
>     }
>
>     private void backtrack(int[] candidates, int remaining, int start,
>                             List<Integer> path, List<List<Integer>> result) {
>         if (remaining == 0) {
>             result.add(new ArrayList<>(path));
>             return;
>         }
>
>         for (int i = start; i < candidates.length; i++) {
>             if (candidates[i] > remaining) {
>                 // Sorted array: no later candidate can fit either.
>                 break;
>             }
>
>             path.add(candidates[i]);
>             // Pass i (not i + 1) since we can reuse the same candidate.
>             backtrack(candidates, remaining - candidates[i], i, path, result);
>             path.remove(path.size() - 1); // backtrack
>         }
>     }
> }
> ```
