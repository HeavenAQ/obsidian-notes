## Summary

Houses are arranged as a **binary tree** rooted at `root`; each node holds an amount of money. A thief cannot rob two **directly-linked** houses (parent and child) on the same night. Return the maximum amount that can be robbed without alerting the police.

**Constraints:** number of nodes in `[1, 10^4]`, `0 <= Node.val <= 10^4`.

**Example 1:** `root = [3,2,3,null,3,null,1]` → `7` (rob 3 + 3 + 1: root, left-right grandchild, right-right grandchild)

**Example 2:** `root = [3,4,5,1,3,null,1]` → `9` (rob 4 + 5)

## Hints

1. Naive recursion ("rob root + grandchildren" vs "skip root, rob children") recomputes the same subtrees exponentially. How is this House Robber, but on a tree?
2. For each node, there are only two states: it is robbed, or it is not. What information does a parent need from each child to decide?
3. Do one post-order DFS that returns a **pair** per subtree: `(best if this node is NOT robbed, best if this node IS robbed)`. Combine children's pairs at the parent — this is DP on trees.

+++ Solution (click to reveal)

**Approach / intuition:** Tree DP via post-order DFS. For each node return two values: `notRobbed` = max loot from this subtree if we skip this node (each child may then be robbed or not — take each child's max), and `robbed` = `node.val` + both children's `notRobbed` (children must be skipped). The answer at the root is `max(notRobbed, robbed)`. Each node is visited once, so there is no repeated work.

**Complexity:** Time `O(n)` (each node visited once), Space `O(h)` recursion stack, worst case `O(n)` for a skewed tree.

```java
/**
 * Definition for a binary tree node.
 * public class TreeNode {
 *     int val;
 *     TreeNode left;
 *     TreeNode right;
 *     TreeNode() {}
 *     TreeNode(int val) { this.val = val; }
 *     TreeNode(int val, TreeNode left, TreeNode right) {
 *         this.val = val; this.left = left; this.right = right;
 *     }
 * }
 */
class Solution {
    public int rob(TreeNode root) {
        int[] res = dfs(root);
        return Math.max(res[0], res[1]);
    }

    // returns {maxIfNotRobbed, maxIfRobbed} for the subtree rooted at node
    private int[] dfs(TreeNode node) {
        if (node == null) return new int[]{0, 0};
        int[] left = dfs(node.left);
        int[] right = dfs(node.right);
        int robbed = node.val + left[0] + right[0];
        int notRobbed = Math.max(left[0], left[1]) + Math.max(right[0], right[1]);
        return new int[]{notRobbed, robbed};
    }
}
```

**Why this works:** The two-state pair captures everything a parent needs about a subtree (the constraint only spans parent–child edges), so local optimal combinations compose into the global optimum — classic DP on trees with O(1) state per node.
