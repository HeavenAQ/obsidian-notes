## Summary

Given two strings `text1` and `text2`, return the length of their **longest common subsequence** (LCS) — a sequence that appears in both strings in the same relative order, but not necessarily contiguously. If there is no common subsequence, return `0`.

**Constraints:** `1 <= text1.length, text2.length <= 1000`. Both strings consist of lowercase English characters only.

**Example 1:** `text1 = "abcde"`, `text2 = "ace"` → `3` (the LCS is `"ace"`).

**Example 2:** `text1 = "abc"`, `text2 = "abc"` → `3`.

**Example 3:** `text1 = "abc"`, `text2 = "def"` → `0` (no common subsequence).

## Hints

1. This is a classic **string-edit style 2D DP**. Think about defining a state on prefixes of both strings: `dp[i][j]` = LCS length using the first `i` characters of `text1` and first `j` characters of `text2`.
2. When `text1[i-1] == text2[j-1]`, that matching character extends the LCS found in the smaller subproblem `dp[i-1][j-1]`. When they differ, the LCS can't use both characters, so take the best of skipping one character from either string.
3. Base case: an empty prefix has LCS length `0`, so `dp[0][j] = dp[i][0] = 0` for all `i, j`. Build the table row by row.

+++ Solution (click to reveal)

**Approach / intuition:** Bottom-up 2D DP over prefixes. Let `dp[i][j]` be the LCS length of `text1[0..i)` and `text2[0..j)`. Recurrence: if `text1.charAt(i-1) == text2.charAt(j-1)`, then `dp[i][j] = dp[i-1][j-1] + 1` (extend the match). Otherwise `dp[i][j] = max(dp[i-1][j], dp[i][j-1])` (drop the last character of whichever string doesn't help). Answer is `dp[m][n]`.

**Complexity:** Time `O(m * n)`, Space `O(m * n)` (can be optimized to `O(min(m, n))` with a rolling array, not required here).

```java
class Solution {
    public int longestCommonSubsequence(String text1, String text2) {
        int m = text1.length(), n = text2.length();
        int[][] dp = new int[m + 1][n + 1];

        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                if (text1.charAt(i - 1) == text2.charAt(j - 1)) {
                    dp[i][j] = dp[i - 1][j - 1] + 1;
                } else {
                    dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1]);
                }
            }
        }

        return dp[m][n];
    }
}
```

**Why this works:** Every optimal alignment of the two strings either matches their last characters (if equal) or discards the last character of one string without losing any potential match (since that character can't be paired with anything better later). The DP explores both options and takes the max, guaranteeing optimal substructure and correctness.
