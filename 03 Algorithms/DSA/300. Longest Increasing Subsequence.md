# Summary

Given an integer array `nums`, return the length of the longest strictly increasing subsequence (elements don't need to be contiguous, but must keep their relative order and each element must be strictly greater than the previous one).

**Constraints:** `1 <= nums.length <= 2500`; `-10^4 <= nums[i] <= 10^4`.

**Examples:**

- `nums = [10,9,2,5,3,7,101,18]` → `4` (one LIS is `[2,3,7,101]`)
- `nums = [0,1,0,3,2,3]` → `4` (one LIS is `[0,1,2,3]`)

## Hints

1. Define `dp[i]` as the length of the longest increasing subsequence that *ends exactly at index *`*i*`.
2. For each `i`, scan every `j < i`: if `nums[j] < nums[i]`, then `dp[i]` could be extended to `dp[j] + 1`.
3. `dp[i] = 1 + max(dp[j])` over all valid `j < i` with `nums[j] < nums[i]` (or just `1` if there's no such `j`). The final answer is `max(dp)`. This gives an O(n²) solution.
4. To go faster (O(n log n)), think about maintaining a separate array `tails`, where `tails[k]` is the *smallest possible tail value* of any increasing subsequence of length `k+1` found so far — and use binary search to place each new number.

### Approach / Intuition

**Core DP formulation (what this course is practicing):** let `dp[i]` = length of the LIS ending at index `i`. Then `dp[i] = 1 + max(dp[j] for j < i where nums[j] < nums[i])`, defaulting to `1` if no such `j` exists. The answer is `max(dp[i])` over all `i`. This is O(n²) time and O(n) space.

**Optimization note:** the O(n log n) "patience sorting" approach maintains a `tails` array where `tails[k]` is the smallest tail value achievable for an increasing subsequence of length `k+1`. For each new number, binary-search `tails` for the first entry `>= num` and overwrite it (or append if `num` is larger than everything seen). The final size of `tails` is the answer. This works because keeping each length's tail as small as possible always leaves the most room to extend later.

**Time complexity:** O(n²) for the DP version shown below (O(n log n) with the binary-search optimization).

**Space complexity:** O(n)

```java
import java.util.Arrays;

class Solution {
    public int lengthOfLIS(int[] nums) {
        int n = nums.length;
        int[] dp = new int[n];
        Arrays.fill(dp, 1);
        int maxLen = 1;
        for (int i = 1; i < n; i++) {
            for (int j = 0; j < i; j++) {
                if (nums[j] < nums[i]) {
                    dp[i] = Math.max(dp[i], dp[j] + 1);
                }
            }
            maxLen = Math.max(maxLen, dp[i]);
        }
        return maxLen;
    }
}
```

**Why this works:** `dp[i]` only ever extends a subsequence ending at some earlier, smaller element, so by the time we compute `dp[i]` every `dp[j]` for `j < i` is already final. Taking the max over all valid extensions guarantees `dp[i]` is the true LIS length ending at `i`, and the max over the whole array is the true answer.

*(For a full O(n log n) implementation, replace the inner loop with a binary search over a *`*tails*`* array as described above — useful as a follow-up once the DP version is solid.)*
