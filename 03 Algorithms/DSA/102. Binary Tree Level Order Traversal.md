## Summary

Given the `root` of a binary tree, return the level order traversal of its nodes' values (i.e., from left to right, level by level).

Constraints: the number of nodes is in the range `[0, 2000]`, and `-1000 <= Node.val <= 1000`.

**Example 1:**
Input: `root = [3,9,20,null,null,15,7]`
Output: `[[3],[9,20],[15,7]]`
Explanation: Level 0 has just `3`; level 1 has `9` and `20`; level 2 has `15` and `7`.

**Example 2:**
Input: `root = [1]`
Output: `[[1]]`

**Example 3:**
Input: `root = []`
Output: `[]`

## Hints

1. "Level by level" is a strong signal for breadth-first traversal rather than depth-first — what data structure naturally processes nodes in the order they were discovered?
2. A plain queue-based BFS visits every node once, but it doesn't inherently know where one level ends and the next begins. How can you separate levels using only the queue's current size?
3. At the start of each iteration of the outer loop, record `queue.size()` — that number is exactly how many nodes belong to the current level, since only nodes from the previous level were enqueued before this point.
4. Dequeue exactly that many nodes, collect their values into a list for this level, and enqueue their (non-null) children as you go. Repeat until the queue is empty.
5. Watch the edge case: if `root` is `null`, return an empty list immediately without touching the queue.

> [!note]- Solution (click to reveal)
> **Approach:** Standard BFS with a "level-size snapshot" trick. Use a queue seeded with the root. On each iteration of the outer while-loop, capture `levelSize = queue.size()` before processing — this is exactly the count of nodes belonging to the current level, because everything currently in the queue was enqueued while processing the previous level. Pop exactly `levelSize` nodes, record their values into a per-level list, and push their non-null left/right children onto the queue (these become next level's nodes). Append the per-level list to the result once the inner loop finishes. Continue until the queue is empty.
>
> **Why this works:** A queue is FIFO, so nodes are dequeued in the exact order they were discovered, which for BFS means strictly level-by-level, left-to-right within a level (since children are enqueued left-then-right). Snapshotting the queue size before the inner loop is what lets us "cut" the continuous stream of dequeues into discrete levels, since new children get appended to the back of the queue and won't be counted in the current level's snapshot.
>
> **Complexity:** Time `O(n)` — every node is enqueued and dequeued exactly once. Space `O(n)` — the queue holds at most one full level (up to `~n/2` nodes for a wide tree), and the result stores every node's value once.
>
> ```java
> import java.util.*;
>
> class Solution {
>     public List<List<Integer>> levelOrder(TreeNode root) {
>         List<List<Integer>> result = new ArrayList<>();
>         if (root == null) {
>             return result;
>         }
>
>         Queue<TreeNode> queue = new LinkedList<>();
>         queue.offer(root);
>
>         while (!queue.isEmpty()) {
>             int levelSize = queue.size();
>             List<Integer> currentLevel = new ArrayList<>(levelSize);
>
>             for (int i = 0; i < levelSize; i++) {
>                 TreeNode node = queue.poll();
>                 currentLevel.add(node.val);
>
>                 if (node.left != null) {
>                     queue.offer(node.left);
>                 }
>                 if (node.right != null) {
>                     queue.offer(node.right);
>                 }
>             }
>
>             result.add(currentLevel);
>         }
>
>         return result;
>     }
> }
>
> // Definition for a binary tree node (LeetCode-provided):
> // class TreeNode {
> //     int val;
> //     TreeNode left;
> //     TreeNode right;
> //     TreeNode() {}
> //     TreeNode(int val) { this.val = val; }
> //     TreeNode(int val, TreeNode left, TreeNode right) {
> //         this.val = val;
> //         this.left = left;
> //         this.right = right;
> //     }
> // }
> ```
