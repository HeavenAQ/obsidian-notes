## Summary

**132. Palindrome Partitioning II** (Medium — Dynamic Programming, string partitioning / interval DP)

Given a string `s`, partition it so that every substring in the partition is a palindrome. Return the **minimum number of cuts** needed for such a partitioning.

**Constraints**: `1 <= s.length <= 2000`; `s` consists of lowercase English letters only.

**Examples**

- `s = "aab"` → `1` (cut into `"aa"`, `"b"`)
- `s = "a"` → `0` (already a palindrome, no cuts needed)
- `s = "ab"` → `1`

## Hints

1. Precompute which substrings of `s` are palindromes so you don't repeatedly re-check the same substring with a naive O(n) expansion each time.
2. Let `dp[i]` = minimum cuts needed for the prefix `s[0..i]`. How does `dp[i]` relate to `dp[j]` for `j < i` when `s[j+1..i]` is itself a palindrome?
3. Build a 2D boolean table `isPalindrome[i][j]` bottom-up: `isPalindrome[i][j] = (s[i] == s[j]) && (j - i < 2 || isPalindrome[i+1][j-1])`, filling it by increasing substring length.
4. `dp[i] = 0` if `s[0..i]` is itself a palindrome; otherwise `dp[i] = min(dp[j-1] + 1)` over every `j` such that `s[j..i]` is a palindrome.

> [!note]+ Solution (click to reveal)
> ### Approach
> 
> Use two DP passes:
> 
> 5. **Palindrome table**: `isPalindrome[i][j]` is true iff `s[i..j]` is a palindrome. Compute it bottom-up by increasing length: a substring is a palindrome if its endpoints match and the inner substring (length - 2) is also a palindrome (or the substring has length 1 or 2).
> 6. **Cut table**: `cuts[i]` is the minimum number of cuts for the prefix `s[0..i]`. If `s[0..i]` is already a palindrome, `cuts[i] = 0`. Otherwise, try every split point `j` (1 <= j <= i): if `s[j..i]` is a palindrome, then `cuts[i]` can be `cuts[j-1] + 1`. Take the minimum over all valid `j`.
> 
> The answer is `cuts[n-1]`.
> 
> **Why this works**: the palindrome table turns an O(n) substring check into O(1) lookup, and the cuts array has optimal substructure — the best partitioning of a prefix is built from the best partitioning of some shorter prefix plus one additional palindromic suffix piece.
> 
> **Complexity**: Time O(n^2) — O(n^2) to fill the palindrome table and O(n^2) worst case to fill the cuts array. Space O(n^2) for the palindrome table (can be optimized to O(n) with more care, but O(n^2) is the standard idiomatic solution).
> 
> ```java
> class Solution {
>     public int minCut(String s) {
>         int n = s.length();
>         boolean[][] isPalindrome = new boolean[n][n];
>         for (int i = 0; i < n; i++) {
>             isPalindrome[i][i] = true;
>         }
>         for (int len = 2; len <= n; len++) {
>             for (int i = 0; i + len - 1 < n; i++) {
>                 int j = i + len - 1;
>                 if (s.charAt(i) == s.charAt(j)) {
>                     isPalindrome[i][j] = (len == 2) || isPalindrome[i + 1][j - 1];
>                 }
>             }
>         }
> 
>         int[] cuts = new int[n];
>         for (int i = 0; i < n; i++) {
>             if (isPalindrome[0][i]) {
>                 cuts[i] = 0;
>                 continue;
>             }
>             cuts[i] = Integer.MAX_VALUE;
>             for (int j = 1; j <= i; j++) {
>                 if (isPalindrome[j][i] && cuts[j - 1] + 1 < cuts[i]) {
>                     cuts[i] = cuts[j - 1] + 1;
>                 }
>             }
>         }
>         return cuts[n - 1];
>     }
> }
> ```
