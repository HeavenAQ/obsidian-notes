## Summary

There are `n` computers numbered `0` to `n - 1`, connected by ethernet cables. You're given `connections`, where `connections[i] = [a, b]` means computers `a` and `b` are directly connected. A computer is considered "connected to the network" if it can reach every other computer, directly or indirectly.

You may take any existing cable that connects two computers and unplug it, then use that same cable to connect any two computers that are not currently directly connected. Return the **minimum number of such operations** needed to make the entire network connected (every computer reachable from every other). If it's impossible, return `-1`.

Constraints:
- `1 <= n <= 10^5`
- `1 <= connections.length <= min(n * (n - 1) / 2, 10^5)`
- `connections[i].length == 2`
- `0 <= a_i, b_i < n`, `a_i != b_i`
- No repeated connections, and no self-loops.

**Example 1:**
Input: `n = 4, connections = [[0,1],[0,2],[1,2]]`
Output: `1`
(Unplug the redundant cable between 1 and 2, and use it to connect computer 3.)

**Example 2:**
Input: `n = 6, connections = [[0,1],[0,2],[0,3],[1,2],[1,3]]`
Output: `2`

**Example 3:**
Input: `n = 6, connections = [[0,1],[0,2],[0,3],[1,2]]`
Output: `-1`
(Only 4 cables for 6 computers — not enough to ever connect them all.)

## Hints

1. To connect `n` computers into one network you need at least `n - 1` cables (a spanning tree). If `connections.length < n - 1`, it's immediately impossible — return `-1` without doing any further work.
2. Otherwise, think about the graph formed by the connections as a set of connected components. Any "extra" cable inside an already-connected component is redundant and free to repurpose.
3. Each operation can merge exactly two components into one. So the answer only depends on how many separate components currently exist.
4. Use Union-Find (or a DFS/BFS pass) to count the number of connected components among the `n` computers. The answer is `components - 1`.

> [!note]- Solution (click to reveal)
> **Approach:** This is a classic "count connected components" problem in disguise.
>
> First, a necessary condition: connecting `n` nodes into a single tree requires at least `n - 1` edges. If we have fewer cables than that, no rearrangement can possibly succeed, so we return `-1` immediately.
>
> If we have enough cables, build a Union-Find (disjoint set) structure over the `n` computers and union every pair given in `connections`. Afterward, count the number of distinct roots — that's the number of connected components, call it `k`. Every cable that lies inside a component beyond what's needed to keep it connected is "spare," and because we already confirmed `connections.length >= n - 1`, there are guaranteed to be enough spare cables to wire the components together. Merging `k` components into 1 always takes exactly `k - 1` operations (each operation joins two components into one), so the answer is `k - 1`.
>
> **Why this works:** a spanning tree over `k` components needs `k - 1` more edges to become one component, and the total edge count being `>= n - 1` guarantees at least that many redundant edges exist somewhere to repurpose (a standard counting/pigeonhole argument), so feasibility and the operation count reduce to the same simple formula.
>
> **Complexity:** Time `O(n + E · α(n))` ≈ `O(n + E)` with path compression and union by rank, where `E = connections.length`. Space `O(n)` for the parent/rank arrays.
>
> ```java
> class Solution {
>     private int[] parent;
>     private int[] rank_;
>
>     public int makeConnected(int n, int[][] connections) {
>         if (connections.length < n - 1) {
>             return -1;
>         }
>
>         parent = new int[n];
>         rank_ = new int[n];
>         for (int i = 0; i < n; i++) {
>             parent[i] = i;
>         }
>
>         for (int[] c : connections) {
>             union(c[0], c[1]);
>         }
>
>         int components = 0;
>         for (int i = 0; i < n; i++) {
>             if (find(i) == i) {
>                 components++;
>             }
>         }
>
>         return components - 1;
>     }
>
>     private int find(int x) {
>         if (parent[x] != x) {
>             parent[x] = find(parent[x]); // path compression
>         }
>         return parent[x];
>     }
>
>     private void union(int x, int y) {
>         int rootX = find(x);
>         int rootY = find(y);
>         if (rootX == rootY) {
>             return;
>         }
>         // union by rank
>         if (rank_[rootX] < rank_[rootY]) {
>             int temp = rootX;
>             rootX = rootY;
>             rootY = temp;
>         }
>         parent[rootY] = rootX;
>         if (rank_[rootX] == rank_[rootY]) {
>             rank_[rootX]++;
>         }
>     }
> }
> ```
