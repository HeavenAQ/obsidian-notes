## Summary

Given a string `s`, find the length of the longest substring without repeating characters.

**Constraints:** `0 <= s.length <= 5 * 10^4`; `s` consists of English letters, digits, symbols, and spaces.

**Example 1:** `s = "abcabcbb"` → `3` (the answer is `"abc"`, with the length of 3).

**Example 2:** `s = "bbbbb"` → `1` (the answer is `"b"`, with the length of 1).

**Example 3:** `s = "pwwkew"` → `3` (the answer is `"wke"`, with the length of 3; note that the answer must be a substring, `"pwke"` is a subsequence and not a substring).

## Hints

1. Brute force would check every substring for uniqueness in `O(n^3)` — think about how to avoid re-scanning characters you've already seen.
2. Maintain a window `[left, right]` that always contains only unique characters; grow `right` one step at a time.
3. Use a hash map (or a fixed-size array for ASCII) from character → last seen index, so when you hit a repeat you can jump `left` directly instead of incrementing it one at a time.
4. When `s[right]` was seen before, only move `left` forward if the previous occurrence is still inside the current window (`left <= lastSeen[s[right]]`) — otherwise it's a stale index outside the window and should be ignored.
5. Track the max window size (`right - left + 1`) as you go; no need to store the substring itself.

> [!note]- Solution (click to reveal)
> **Approach:** Sliding window with a hash map recording the most recent index of each character seen so far. Expand the window by moving `right` through the string. If the character at `right` was seen before *and* its last position is within the current window (`>= left`), shrink the window by moving `left` to `lastSeen[c] + 1`. After each step, update the map and the best length using `right - left + 1`.
>
> **Why this works:** The window `[left, right]` is an invariant that always holds only distinct characters. Whenever a duplicate would enter the window, we jump `left` past the previous occurrence in one step rather than incrementing it repeatedly, which keeps the whole scan to a single pass over `s`.
>
> **Complexity:** Time `O(n)` — each index is visited by `right` once and `left` moves forward monotonically, so total work is linear. Space `O(min(n, alphabet size))` for the hash map.
>
> ```java
> class Solution {
>     public int lengthOfLongestSubstring(String s) {
>         Map<Character, Integer> lastSeen = new HashMap<>();
>         int maxLen = 0;
>         int left = 0;
>
>         for (int right = 0; right < s.length(); right++) {
>             char c = s.charAt(right);
>
>             if (lastSeen.containsKey(c) && lastSeen.get(c) >= left) {
>                 left = lastSeen.get(c) + 1;
>             }
>
>             lastSeen.put(c, right);
>             maxLen = Math.max(maxLen, right - left + 1);
>         }
>
>         return maxLen;
>     }
> }
> ```
