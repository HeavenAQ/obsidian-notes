## Summary

You are given an integer array `coins` representing coins of different denominations and an integer `amount`. Return the **fewest number of coins** needed to make up that amount. If the amount cannot be made up by any combination of the coins, return `-1`. You have an infinite supply of each coin.

**Constraints:** `1 <= coins.length <= 12`, `1 <= coins[i] <= 2^31 - 1`, `0 <= amount <= 10^4`.

**Example 1:** `coins = [1,2,5]`, `amount = 11` → `3` (5 + 5 + 1)

**Example 2:** `coins = [2]`, `amount = 3` → `-1`

## Hints

1. Greedy (always taking the largest coin) fails — try `coins = [1,3,4]`, `amount = 6`. Think about sub-problems instead.
2. Define `dp[i]` = minimum coins needed to make amount `i`. What is `dp[0]`? How does `dp[i]` relate to `dp[i - coin]`?
3. This is an **unbounded knapsack**: each coin can be reused, so iterate amounts from `1` to `amount` and try every coin at each amount.

+++ Solution (click to reveal)

**Approach / intuition:** Bottom-up 1D DP over amounts. `dp[i]` is the fewest coins to form amount `i`. For each amount `i`, try every coin `c <= i` and take `dp[i] = min(dp[i], dp[i - c] + 1)`. Initialize `dp[0] = 0` and everything else to a sentinel (`amount + 1`, effectively infinity). Because coins are reusable, iterating each amount over all coins naturally allows repeats (unbounded knapsack).

**Complexity:** Time `O(amount × coins.length)`, Space `O(amount)`.

```java
import java.util.Arrays;

class Solution {
    public int coinChange(int[] coins, int amount) {
        int[] dp = new int[amount + 1];
        Arrays.fill(dp, amount + 1); // sentinel: unreachable
        dp[0] = 0;
        for (int i = 1; i <= amount; i++) {
            for (int coin : coins) {
                if (coin <= i) {
                    dp[i] = Math.min(dp[i], dp[i - coin] + 1);
                }
            }
        }
        return dp[amount] > amount ? -1 : dp[amount];
    }
}
```

**Why this works:** Every optimal way to form amount `i` ends with some last coin `c`; removing it leaves an optimal solution for `i - c` (optimal substructure). The DP considers all possible last coins, so it finds the global minimum.
