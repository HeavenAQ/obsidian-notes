## Summary

A message containing letters `A`-`Z` can be encoded into numbers using the mapping `'A' -> "1"`, `'B' -> "2"`, ..., `'Z' -> "26"`. Given a string `s` containing only digits, return the **number of ways** to decode it.

**Constraints:**

- `1 <= s.length <= 100`
- `s` contains only digits and may contain leading zeros.

**Examples:**

- Input: `s = "12"` → Output: `2` (decodes to `"AB"` or `"L"`)
- Input: `s = "226"` → Output: `3` (decodes to `"BZ"`, `"VF"`, or `"BBF"`)
- Input: `s = "06"` → Output: `0` (a leading zero can never be decoded alone)

## Hints

1. This has the same shape as Climbing Stairs — define `dp[i]` = number of ways to decode the first `i` characters of `s`.
2. `dp[i]` can come from `dp[i-1]` (if the single digit `s[i-1]` is valid, i.e. `1`-`9`) and/or from `dp[i-2]` (if the two-digit number `s[i-2..i-1]` is between `10` and `26`).
3. Watch out for `'0'`: it is never valid as a standalone digit, only as the second digit of `"10"` or `"20"`.
4. You only ever need the last two `dp` values, so this can be optimized to O(1) extra space.

> [!note]+ Solution (click to reveal)
> ### Approach
> 
> Let `dp[i]` be the number of ways to decode the prefix `s[0..i)`. Base cases: `dp[0] = 1` (empty prefix, one trivial way) and `dp[1] = 1` if `s[0] != '0'` else `0`.
> 
> For `i` from `2` to `n`:
> 
> - If the single digit `s[i-1]` is `1`-`9`, add `dp[i-1]` to `dp[i]` (treat it as its own letter).
> - If the two-digit number formed by `s[i-2..i-1]` is `10`-`26`, add `dp[i-2]` to `dp[i]` (treat the pair as one letter).
> 
> The answer is `dp[n]`. Since each step only looks back one and two positions, the array can be collapsed into two rolling variables.
> 
> **Why this works:** every position in the string is either consumed as a single valid digit (extending a decoding that ended one character earlier) or consumed together with the previous digit as a valid two-digit letter (extending a decoding that ended two characters earlier). Summing both possibilities counts every valid decoding exactly once, with no double counting, because the two cases correspond to different lengths of the last "chunk".
> 
> **Complexity:** Time O(n), Space O(1) (rolling variables instead of a full `dp` array).
> 
> ```java
> class Solution {
>     public int numDecodings(String s) {
>         int n = s.length();
>         if (n == 0 || s.charAt(0) == '0') return 0;
> 
>         int prev2 = 1; // dp[i-2], starts as dp[0]
>         int prev1 = 1; // dp[i-1], starts as dp[1]
> 
>         for (int i = 2; i <= n; i++) {
>             int current = 0;
>             int oneDigit = s.charAt(i - 1) - '0';
>             int twoDigit = Integer.parseInt(s.substring(i - 2, i));
> 
>             if (oneDigit >= 1 && oneDigit <= 9) {
>                 current += prev1;
>             }
>             if (twoDigit >= 10 && twoDigit <= 26) {
>                 current += prev2;
>             }
> 
>             prev2 = prev1;
>             prev1 = current;
>         }
> 
>         return prev1;
>     }
> }
> ```
