## Summary

A robot sits at the top-left corner of an `m x n` grid. It can only move **down** or **right** at each step, and it wants to reach the bottom-right corner. Return the number of distinct paths from the top-left to the bottom-right corner.

**Constraints:** `1 <= m, n <= 100`; the answer is guaranteed to fit in a 32-bit integer.

**Examples:**

- `m = 3, n = 7` → `28`
- `m = 3, n = 2` → `3` (paths: Right→Down→Down, Down→Right→Down, Down→Down→Right)

## Hints

1. Think recursively: how many ways are there to reach cell `(i, j)`, in terms of the cells right above it and directly to its left?
2. Since the robot can only arrive at `(i, j)` from `(i-1, j)` (above) or `(i, j-1)` (left), the number of paths to `(i, j)` is the *sum* of the paths to those two cells.
3. The entire first row and first column only have exactly one path each (a straight line), which gives you your base case.
4. Once you have the 2D recurrence working, notice that row `i` only ever depends on row `i-1` — you can collapse the DP table to a single 1D array of size `n` and update it in place.

### Approach / Intuition

This is the canonical grid-DP problem. Let `dp[i][j]` be the number of unique paths from `(0,0)` to `(i,j)`. Because the robot only moves right or down, a path can only enter `(i,j)` from directly above or directly to the left, so:

`dp[i][j] = dp[i-1][j] + dp[i][j-1]`

with `dp[0][j] = 1` for all `j` and `dp[i][0] = 1` for all `i` (only one way to walk in a straight line along the top row or left column). The answer is `dp[m-1][n-1]`.

Since row `i` only depends on row `i-1`, we can compress the table to a single rolling array `dp` of length `n`, updating `dp[j] += dp[j-1]` left to right for each row.

(There is also a closed-form combinatorial answer, `C(m+n-2, m-1)`, since any path is a sequence of `m-1` "down" moves and `n-1` "right" moves in some order — but the DP formulation is what this course is practicing.)

**Time complexity:** O(m · n)

**Space complexity:** O(n) with the 1D rolling array

```java
import java.util.Arrays;

class Solution {
    public int uniquePaths(int m, int n) {
        int[] dp = new int[n];
        Arrays.fill(dp, 1);
        for (int i = 1; i < m; i++) {
            for (int j = 1; j < n; j++) {
                dp[j] += dp[j - 1];
            }
        }
        return dp[n - 1];
    }
}
```

**Why this works:** Before the inner loop updates index `j`, `dp[j]` still holds last row's value, i.e. `dp[i-1][j]`. `dp[j-1]` has already been updated for the current row (since we iterate `j` left to right), i.e. it holds `dp[i][j-1]`. So `dp[j] += dp[j-1]` computes exactly `dp[i][j] = dp[i-1][j] + dp[i][j-1]` in place, one row at a time.
